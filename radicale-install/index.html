Radicaleを導入してGoogle連絡先から移行

2026-03-09

最近セルフホストが楽しい

この頃自宅サーバーにあれこれウェブサービスをdockerで入れるのがマイブームで、もはや手段と目的が反転してなにかしらセルフホストできるものはないかと調査したりもしてる。今回は、CalDAVというプロトコルでカレンダーとToDo、CardDAVというプロトコルで連絡先を管理するRadicaleのサーバーをホスティングした。

CalDAV、CardDAVの同期はローカルデータが保存されるため、家の外で繋がらない(繋げるのが大変)というセルフホスティングサービスの弱点にあまり影響されないのも大変良い。

Radicaleの導入

インストール

Radicaleのドキュメントにリンクが貼られてるcompose.yamlを落として'docker compose up -d'してもよいが、私はいくつか修正している

services:
  radicale:
    image: ghcr.io/kozea/radicale:stable
    container_name: radicale
    ports:
      - 5313:5232
    volumes:
      - ./config:/etc/radicale
      - ./data:/var/lib/radicale
    restart: unless-stopped
    logging:
      driver: "json-file"
      options:
        max-size: "5m"
        max-file: "1"

変更点の意図は以下の通りだ。

  • ports: 我が家では5300から順番にポートを当てているため、ポート修正は必須だ
  • volumes: 本家ではvolumeを別途定義しているが、個人のシンプルな使い方ならvolumesで直接相対パスで
  • restart: サーバー再起動で自動で立ち上がるように
  • logging: ログの容量制限

ユーザー生成

RadicaleのユーザーはWebUIからは登録できず、CLIで生成しなければならない。

まずは「パスワード認証を使う」という設定を、コンテナ内における/etc/radicaleディレクトリーに作る。上のcomposeファイルでいうと./configに当たる。次の内容で./config/configというファイルを生成する。

type = htpasswd
htpasswd_filename = /etc/radicale/users
htpasswd_encryption = bcrypt

/etc/radicale/usersというのは無難に決めたファイル名だ。次に、このusersファイルを作る。場所は/etc/radicale/なので、configと同じディレクトリーになる。usersの中身は

ユーザー名:パスワード
ユーザー名:パスワード
ユーザー名:パスワード
...

と、利用したいユーザー情報を列挙する。ただし、パスワードは平文ではなくハッシュ値を生成しなければならない。htpasswdというコマンドで作れるそうだが、私はPythonで生成した。

import bcrypt

password = "veryverystrongpassword"  # 適宜変更

hashed = bcrypt.hashpw(password.encode("utf-8"), bcrypt.gensalt())
print(hashed.decode("utf-8"))

これで出力されるハッシュ値と好きなユーザー名をusersに書き込む。

権限指定

Radicaleのdockerイメージ内ではRadicaleがUID:GID=1000:1000となっているようで、今の設定のままdockerを起動すると権限エラーで立ち上がらない。configとdataはcompose upする前に作るか、権限変更をする。

sudo chown -R 1000:1000 config data

これで動作するはずだ。

HTTPS化

CalDAVサーバーとしてはこれで動作するが、iOSやMacOSで使うためにはHTTPS化、それも自己署名ではない、外部認証局によるHTTPS化が必要だ。HTTPや自己署名HTTPSだとiOSのアカウント追加で認証エラーになって進めない。私は以前自前のサーバーをLet's EncryptでHTTPS化したため、今回追加で何かすることはなかった。そのうちHTTPS化の記事も書かないとと思ってる。

次のMac連携はHTTPS化できてる前提で進める。

Mac連携

iOSもMacOSもやり方はそう変わらない。インターネットアカウントの追加メニューで、CalDAVとCardDAVそれぞれに今立ち上げたRadicaleサーバーのURL、ユーザー名、パスワードを入れるだけだ。

ログインできれば、CalDAVではカレンダーとリマインダー、CardDAVでは連絡先の同期が取れるようになるし、カレンダーアプリやリマインダーアプリを立ち上げるとRadicaleアカウントが登録されていることを確認できる。

Googleから連絡先の移行

私は予定とTodoは量が少なくて手打ち移行できたが、連絡先は手打ちでは大変すぎる。しかし、Google連絡先をエクスポートしたファイルをそのままアップロードということもできず、少し手間取った。

連絡先のエクスポート

Google連絡先のWeb画面からダウンロードする。エクスポートメニューから「Android または iOS 用 vCard」を選ぶと.vcfファイルでダウンロードされる。そのファイルを、RadicaleのWebUI(下の画像)の右下にある、いかにもアップロードボタンのように見える青い丸をクリックしてアップロードすれば済みそうに思えるが、そうはいかない。

なぜかJavascriptエラーになってアップロードできない。Firefox、Safari、Edgeのいずれのブラウザでもできなかった。別の方法を取る必要がある。

下準備

まず下準備として、アップロード先の連絡先ディレクトリーを作成する。RadicaleではTodoのグループやカレンダー、連絡先をそれぞれ個別のディレクトリーで管理するため、アップロードする前にファイルを入れる「箱」を用意する必要がある。上のメインUIの右下の緑の丸(+と書いてある)を押してTypeとしてAddress bookを選んで生成する。

そうすれば次のように空の連絡先が生成され、紐づいたURLが表示される。このURLがアップロード先となる。

.vcfファイルの加工

HTTPリクエストを用いて連絡先をRadicaleにアップロードするときは、連絡先1つにつき1ファイルである必要があり、Googleからダウンロードした.vcfファイルを個別ファイルに分割しなければならない。

また、Googleの.vcfファイルの各連絡先にはUID欄がないが、Radicaleは連絡先にUIDがないとエラーになって登録できない。

大変なので、AIに頼んでオリジナルの.vcfファイルを分解し、連絡先1つずつファイルとして生成してHTTPリクエストを送るPythonスクリプトを作ってもらった。次のスクリプトで、URL、USER、PASS、ORIGINAL_FILEを環境に合わせて書き換えて実行すればよい。

import re
import subprocess
import uuid

URL = "上で作った連絡先のURL"
USER = "Radicaleユーザー名"
PASS = "パスワード"
ORIGINAL_FILE = "Googleからダウンロードした.vcfファイルのパス"

text = open(ORIGINAL_FILE, encoding="utf-8-sig").read()
cards = re.findall(r"BEGIN:VCARD.*?END:VCARD", text, re.DOTALL)

for card in cards:
    uid = str(uuid.uuid4()).upper()  # ランダムUIDを生成
    card = card.replace("END:VCARD", f"UID:{uid}\nEND:VCARD")  # 連絡先に追加

    path = f"./{uid}.vcf"
    open(path, "w").write(card)  # UID.vcfで個別ファイル生成

    # サーバーへ転送
    result = subprocess.run(
        [
            "curl",
            "-s",
            "-o",
            "/dev/null",
            "-w",
            "%{http_code}",
            "-u",
            f"{USER}:{PASS}",
            "-X",
            "PUT",
            "-H",
            "Content-Type: text/vcard; charset=utf-8",
            "-T",
            path,
            f"{URL}{uid}.vcf",
        ],
        capture_output=True,
        text=True,
    )
    code = result.stdout.strip()
    # エラーになったファイルの告知
    if code not in ("200", "201", "204"):
        print(f"[FAIL {code}] {uid}")

print(f"{len(cards)} 件完了")

完了したあとRadicaleのWebUIをリフレッシュすれば、エラーがでてなければ連絡先が増えているはずだ。

単純ファイルコピーでもできるが

Radicaleは連絡先を単にファイルとして保持してるだけなため、HTTPリクエストを使わず、生成した個別ファイルを単純コピーしてしまうという手もある。コピー先の場所は以下の通りだ。

Dockerディレクトリー/data/collections/collection-root/ユーザー名/連絡先のUUID(URLの末尾)

HTTPリクエストより単純コピーの方が高速だしシンプルだが、この方法だと万が一正常に登録されてない連絡先があったときどれなのか調べるのが大変になるので、HTTPリクエストの方が安全だろう。
