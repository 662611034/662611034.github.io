Enumは便利だけど手間がかかる

Rustを書くときenumをかなり頻繁に使う。引数が取りうる値を型で制限できるから安全だし、文字列リテラルじゃないから補完が効くし、名前がついてるから分かりやすい。ChatGPTに聞いてみたところ、入力が2種類の場合でも「Yes / No」での判定が微妙なケースではenumを使った方がいいとか。

しかし、1点面倒なことがある。enumはiter()が使えない。例えばGUIでコンボボックスを作るとき

egui::ComboBox::from_label("Select one!")
    .selected_text(format!("{:?}", selected))
    .show_ui(ui, |ui| {
        // こんなことはできない
        MyEnum.for_each(|variant|
            ui.selectable_value(&mut selected, variant, variant.as_str());
        );
    }
);

みたいなことができれば幸せになれるんじゃないかと思うことが多々ある。まあ、今どきのIDE、エディタは便利な編集機能がいろいろあるので、1～2回するぐらいなら手動で全然間に合うし、大量で書かなきゃならないようなときは、専用のクレイトを使えばいいし、特に困りはしないのだが、ふと「文字列操作でマクロを自動生成すれば、楽しいんじゃないかな」（"便利"ではない）と思い立ったので、作ってみた。

use std::io::Read;

fn variants_iter(definition: &str) -> impl Iterator<Item = String> {
    definition
        .split('\n')
        .map(|line| line.trim())
        .filter(|&line| {
            !(line.contains("{") || line.contains("}") || line.is_empty() || line.starts_with("//"))
        })
        .map(|s| s.replace(",", ""))
}

fn get_enum_name(definition: &str) -> Option<String> {
    let mut flag = false;
    for word in definition
        .split('\n')
        .map(|line| line.trim())
        .filter(|&line| line.contains("{") && line.contains("enum"))
        .next()
        .unwrap_or("")
        .replace("{", "")
        .trim()
        .split_whitespace()
    {
        if word.trim().is_empty() {
            continue;
        }
        if word == "enum" {
            flag = true;
            continue;
        }
        if flag {
            return Some(String::from(word));
        }
    }
    None
}

fn gen_enum_iter(definition: &str, macro_name: &str) -> Option<String> {
    let enum_name = get_enum_name(definition)?;
    let mut description = String::from("macro_rules! ");
    description += &macro_name;
    description += " {\n()=>{\n[\n";
    variants_iter(definition)
        .for_each(|variant| description += &format!("{}::{},\n", enum_name, variant));
    Some(description + "]\n.iter()\n};\n}\n")
}

fn gen_vec_mapping(definition: &str, macro_name: &str) -> Option<String> {
    let mut description = String::from("macro_rules! ");
    description += &macro_name;
    description += " {\n($self:expr => $table:ident) => {\nmatch $self {\n";

    variants_iter(definition)
        .enumerate()
        .for_each(|(idx, variant)| description += &format!("Self::{variant} => $table[{idx}],\n"));
    Some(description + "}\n};\n}")
}

fn gen_as_str(definition: &str, macro_name: &str) -> String {
    let mut description = String::from("macro_rules! ");
    description += &macro_name;
    description += " {\n($self:expr) => {\nmatch $self {\n";
    variants_iter(definition).for_each(|variant| {
        description += &format!("Self::{variant} => \"{variant}\",\n");
    });
    description + "}\n};\n}"
}

fn main() {
    let mut definition = String::new();
    std::io::stdin()
        .read_to_string(&mut definition)
        .expect("Failed to read from stdin");

    match get_enum_name(&definition) {
        Some(_) => {}
        None => panic!("enumの名前の取得に失敗"),
    };

    println!("/*\n");
    println!("{}\n", gen_enum_iter(&definition, "enum_iter").unwrap());
    println!("{}\n", gen_vec_mapping(&definition, "enum_map").unwrap());
    println!("{}\n", gen_as_str(&definition, "enum_str"));
    println!("*/");
}

なんてことはない。enumの定義を標準入力でなげると、文字列操作してマクロを作って出力する。ターミナルで

head -n 50 main.rs | tail -n 6

みたいな感じでenum定義部分を投げればマクロが出来上がるという想定だ。

コマンドライン引数でどのマクロを選ぶかとマクロ名なんかを指定できるようにしてもいいが、手間を考えると張り付けてから消すなり書き換えるなりする方が多分速いと思ったから「問答無用で全部出力」するようにしてる。

コメントや、要素を持つenumには対応できないが、そういうenumはどのみち手動で細かな作業が必用になるから、別にこれでいいだろうと考えている。
