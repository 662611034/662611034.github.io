Misskeyで削除したユーザー名を再利用する

2026-02-28

misskeyでは消したユーザーの名前が再利用できない

自宅サーバーでMisskeyを試してたところ、動作テスト用に作ったtestやtest1というアカウントを削除したあと、同じ名前で再登録しようとしたら「利用できない」とエラーが発生した。どうやら、なりすまし防止のためか、消したユーザーを保存しているようだ。セルフホストする側からしたら迷惑な話である。

削除済みユーザーの消し方

以下のコマンドはMisskeyのドキュメント通り、ほぼデフォルトのままインストールしたケースを想定したものである。

まずDockerのコンテナ経由でPostgreSQLに接続する。

docker compose exec db psql -U example-misskey-user misskey

もし.config/default.ymlでDBユーザー名を変えた場合example-misskey-userのところは変更したDBユーザー名に読み替える。

DBにアクセスできたら削除済みユーザーリストを確認できる。

SELECT * FROM used_username;

全削除する場合は

DELETE FROM used_username;

個別削除は適宜AIに相談すればすぐ見つかると思う。公開サーバーでやるときはくれぐれも気を付けるように。
